To compile and run this multi-process system, you need to link specific libraries—such as pthreads for the server threads and rt (real-time) for shared memory and semaphores. [1, 2] 
## 1. Manual Compilation
Use gcc to compile the server and client separately. Note the order: the -lpthread and -lrt flags should typically come at the end of the command. [1, 3] 

* Compile the Server:

gcc server.c -o server -lpthread -lrt

* Compile the Client:

gcc client.c -o client -lrt

[1] 

## 2. Automating with a Makefile
A Makefile is the most efficient way to "club" these together for development. It ensures that only changed files are recompiled. [4, 5] 
Create a file named Makefile in your directory:

CC = gcc
CFLAGS = -Wall -g
LDFLAGS = -lpthread -lrt

all: server client

server: server.c common.h
	$(CC) $(CFLAGS) server.c -o server $(LDFLAGS)

client: client.c common.h
	$(CC) $(CFLAGS) client.c -o client -lrt

clean:
	rm -f server client

Simply type make in your terminal to build both programs simultaneously. [4] 
## 3. Execution "Clubbing"
Since these are separate processes that communicate, you must run the Server first so it can initialize the shared memory and message queues. [6] 

* Option A: Multiple Terminals (Recommended)
Open two terminal windows.
1. In Terminal 1: ./server
   2. In Terminal 2: ./client
* Option B: Background Execution
Run the server in the background and then launch the client in the same terminal:

./server & ./client

[7, 8] 
* Option C: Automated Script
Create a run.sh script to handle the startup and cleanup:

#!/bin/bash
./server &       # Start server in background
SERVER_PID=$!    # Save its process ID
sleep 1          # Give server a second to initialize
./client         # Run client in foreground
kill $SERVER_PID # Kill server when client exits


Note: Always remember to use ipcs to check for lingering IPC resources and ipcrm to manually remove them if your program crashes. [9]
